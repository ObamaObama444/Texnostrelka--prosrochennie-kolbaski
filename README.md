# BookVerse Case

Решение кейса «Умный поиск по книгам» для направления «Искусственный интеллект».

Сервис умеет:
- загружать книги пользователя в формате `.txt` и дополнительно `.fb2`,
- находить 3-5 наиболее подходящих фрагментов по текстовому запросу,
- отвечать на вопросы по книге в режиме RAG: сначала поиск фрагментов, потом grounded-ответ только по найденным цитатам,
- честно сообщать, если точного ответа в тексте нет или запрос, вероятно, не связан с содержанием книги.

## Стек технологий

- `Python 3.11+`
- `FastAPI` и `Uvicorn`
- `SQLite` для хранения книг, статусов и фрагментов
- `FAISS` для векторного поиска
- `BM25 + embeddings + hybrid ranking` для retrieval
- `Mistral AI`:
  - `mistral-embed` для embeddings
  - `mistral-small-latest` для ответов по найденным цитатам
- фронтенд без Node-сборки: HTML, CSS, JavaScript

## Архитектура решения

Сервис работает по схеме:

1. Пользователь загружает книгу.
2. Книга разбирается на главы и фрагменты.
3. Фрагменты индексируются:
   - лексически через BM25,
   - семантически через embeddings и FAISS.
4. По запросу поиска сервис возвращает несколько наиболее релевантных фрагментов с указанием источника.
5. По вопросу сервис сначала ищет фрагменты, затем передает только найденные цитаты в Mistral и получает grounded-ответ.
6. Если релевантной опоры недостаточно, сервис не выдумывает ответ и сообщает об этом пользователю.

## Что реализовано

- загрузка собственных книг через веб-интерфейс,
- индексация книг в фоне,
- статусы книг: `PROCESSING`, `READY`, `ERROR`,
- поиск фрагментов по одной книге или по нескольким книгам,
- ответы по книге с цитатами,
- reader-режим с переходом к месту в книге,
- предупреждения для слабых совпадений и нерелевантных запросов,
- тесты на ingestion, retrieval и API-сценарии.

## Как запустить сервис

### 1. Перейти в папку проекта

```bash
cd "/Users/galagozaevgenij/Documents/технострелка че/BookVerse"
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -e .[dev]
```

### 4. Настроить Mistral

Создайте локальный `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните переменные:

```env
MISTRAL_API_KEY=your_key_here
MISTRAL_BASE_URL=https://api.mistral.ai/v1
MISTRAL_CHAT_MODEL=mistral-small-latest
MISTRAL_EMBED_MODEL=mistral-embed
MISTRAL_TIMEOUT=60
```

Важно:
- `.env` не должен попадать в git,
- если `MISTRAL_API_KEY` не задан, поиск фрагментов продолжит работать, но режим ответов по книге будет недоступен.

### 5. Запустить сервис

Если порт `8000` свободен:

```bash
uvicorn bookverse.main:app --host 127.0.0.1 --port 8000
```

Если `8000` занят другим приложением, используйте:

```bash
uvicorn bookverse.main:app --host 127.0.0.1 --port 8001
```

После запуска откройте:

- `http://127.0.0.1:8000`
- или `http://127.0.0.1:8001`

в зависимости от выбранного порта.

## Как загрузить книги

Вариант через интерфейс:

1. Открыть главную страницу сервиса.
2. Нажать кнопку добавления книги.
3. Выбрать файл `.txt` или `.fb2`.
4. Дождаться смены статуса книги на `Готово`.

Поддерживаемые форматы:
- обязательный по ТЗ: `.txt`
- дополнительный: `.fb2`

## Интерфейс

Сервис реализован как веб-приложение, которым можно пользоваться без дополнительных консольных команд после запуска.

Основные сценарии:
- библиотека книг,
- поиск фрагментов,
- ответы по книге,
- просмотр содержимого книги и переход к месту из найденной цитаты.

## Тестирование

Запуск тестов:

```bash
pytest -q
```

Что проверяется:
- парсинг `.txt` и `.fb2`,
- корректный чанкинг по главам,
- API-сценарий `upload -> search -> ask -> delete`,
- честный отказ, если ответа в книге нет.

## Ключевые API-эндпоинты

- `GET /api/status` — статус сервиса и провайдера
- `GET /api/books` — список книг
- `POST /api/books/import` — загрузка книги
- `DELETE /api/books/{book_id}` — удаление книги
- `GET /api/books/{book_id}/content` — содержимое книги для reader-режима
- `POST /api/search` — поиск фрагментов
- `POST /api/ask` — ответы по книге с цитатами

## Почему решение соответствует ТЗ

- есть работа с книгами в формате `.txt`,
- есть возможность загрузки своих книг,
- поиск возвращает несколько наиболее подходящих фрагментов,
- для каждого фрагмента выводится источник,
- ответы строятся по найденным цитатам,
- при отсутствии надежной опоры сервис честно отказывает,
- есть удобный веб-интерфейс для членов жюри.

## Ограничения текущей версии

- для режима вопросов требуется настроенный ключ Mistral,
- `.fb2` поддерживается как дополнительный формат, хотя в ТЗ обязателен только `.txt`,
- качество поиска зависит от релевантности найденных фрагментов и может ухудшаться на очень коротких или заведомо нерелевантных запросах, однако интерфейс помечает такие случаи как слабые совпадения.
